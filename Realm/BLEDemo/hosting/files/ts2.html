If we tell MongoDB we want to store timeseries data - it can optimise it's storage and retrieval.

db.createCollection("readings",{timeseries:{"timeField":"timestamp","metaField":"sensor"}})

MongoDB stores data for the same device together, sharing a "bucket" document and a disk block

Each block fills to a max size then a new one is created.

Only blocks need indexes so indexes are much smaller.

Each block covers data in a time period.

Fetching data for device an time is fewer reads.

Also as data is stored together for a given device+time, compression is much better

This is 3-55X faster to Read and uses 3-5X less space on disk.

This will improve over time.
